CSS Tutorial

Introduction

Hello and welcome to the CSS Tutorial page.

This is a basic tutorial about CSS and how we can use different css settings to accomplish functionality and create visibly pleasing content.

Before we begin with our CSS tutorial, we added some basic html elements to the page. These elements are some of the most common found in html pages.

We are assuming that you already have some familiarity with html tags and how they are nested within each other.

In this page, we have added some basic headers, nav tags, links, paragraphs and general structural tags.

At this point, this page is completely stripped of CSS and the only thing that renders is the lines of text that are being typed within the html document.

As we progress through this tutorial, we will be adding new CSS rules and we will be moving things around in order to make our page more pleasing to view and more functional.

A brief history of HTML/CSS

In the early days of the world wide web (WWW), documents or other information were often sent as simple plain text across the wires that formed the internet.
Because the original intent was to allow scientists to share information at the particle accelerator CERN, Sir Tim Berners-Lee, the inventor of the WWW, developed the Hyper Text Markup Language (HTML).
HTML included the means to allow some basic formatting of the documents being shared.

Once styling became more complex and demanding, there was a need for a way that would allow the separation of the code that related to the styling of the page from the rest of the document.

CSS was introduced in order to satisfy this need. CSS files are used to control the attributes of a webpage, while the overall style of an html page is being managed through separate files.

By including some type of order and precedence we can keep CSS code DRY (don't repeat yourself). This increases readability for the developer while reducing the memory requirements on both the client and the server side, as well as during the transmission process.

How was CSS used on this page?

Even though there is no CSS applied to the page, you can see that some formatting is already applied to it.

This formatting comes from the browser's preset settings and it might be different from how other browsers format the page.

That is why you see things like:

  1. The <h1> and <h2> tags of the page in a larger font size.
  2. The fonts of the page being in a specific preset font format.
  3. Some indentation between the text and the window's frame.
  4. Spacing between the different paragraphs and the headers of the page.

As you progress through the pages of this tutorial, you will be able to see the css code that we added by viewing the things entered in the <code> tag.

For example, you will see code displayed like this:

h1 {
font-size: 3.2rem;
line-height: 5.2rem;
font-family: Arial;
font-weight: 600;
}
