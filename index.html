What is CSS?

CSS stands for Cascading Style Sheets. CSS is a style language sheet that is being used to modify the way an HTML page is being presented. Let's take a random person as an example. This person might wear a coat, a skirt, and a hat one day, and jeans, a t-shirt and glasses the other day. The person is always the same but the way she looks is different based on the clothes she is wearing. Likewise, CSS can give a different look, feel and level of interactivity to the different components of the HTML page, based on the different properties we are modifying to the elements (tags) of the HTML page.

Syntax structure

CSS has a syntax that has some similarities to the way JSON files or JavaScript object literals are structured. The main components of CSS are the following:

Selector

The selector is the one that points to the HTML element (or elements) that you want to style. If this would be stated in plain English, it would read like: I want you to select the h1 tags.

Declarations

These are the different properties that state the type of styling you want a particular element to have and what the value is. If this would be stated in plain English, it would read something like: I want to select the h1 tags. (selector) and I want their color to be green. --> h1{ color: green; }.
Notice how the declaration is siting between two curly braces. All the declarations of a selector are included between two curly braces and they are separated by semi-colons (;).

Property

The property is the entity that states what needs to change (eg: color, positioning, background, display). It is the first part of the declaration, before the colon symbol (:) --> h1{ color: <-- The property is the one to the left; }

Value

The value is the entity that states how the property changes / what the value of the property is. --> h1 { The value is the one to the right --> green; }

Comments

Comments is CSS are created by using a forward slash and a star to open them, and a star followed by a forward slash to close them. Comments can be a convenient way to keep notes about edits that are being done in CSS or temporarily de-activate declarations in order to observe the behavior of the page without deleting the declaration, in the event you will need to reactivate it again. /*Hello, I am a comment. Use me to keep notes in CSS*/

Example

p{ color: blue; /*font-family: verdana;*/ /*font-size: 20px;*/ }

Syntax Elements

The CSS syntax elements are:

  1. Selector
  2. Declarations
  3. Property
  4. Value

Syntax elements distinction

  • All the declarations for a selector are included within curly braces ({ })
  • Each declaration is separated by a semicolon.
  • Declaration properties are separated by its values with a colon.

CSS comments

The comments in CSS can be entered with slashes and asterisks. You can open a comment with a slash and an asterisk (/*) and you can close it with an asterisk and a slash (*/).

All the different popular browsers come in the market with some predefined format for several of the primary HTML elements. That happens because in the past, when CSS was not that widely used, the web pages were more static and there was uniformity in some key components of the pages. Hence, the browsers come with some format for all the h tags, there is usually some predefined padding and margin and finaly, the font-size and font-family is predefined.

Things get more complicated because different browsers are handling the above and several other properties in a different way. Therefore, when the web developer creates a new CSS environment, unexpected results might render in specific browsers, if compared to other ones. Hence the web developer cannot accomplish uniformity in the online experience.

In order to resolve this challenge, the developers go through a process of normalizing some or all of the styles that are being affected. There are several different flavors of CSS reset that you can chose from. Some of them are very detailed and extensive while others are barebones simple. Depending on your level of knowledge and comfort, you can go with any way. My suggestion is to start with the absolute basics first and then start thinking if you want to reset more things. In that way, you will avoid any unexpected browser behavior and you will not have to rebuild the CSS from the ground up and for elements that you were not planning to change anyway. An example could be the reset of the li elements in ordered and unordered lists. If you are fine with the way the li elements are rendering, then you should not go ahead and reset them because then nothing will render with a bullet or a numbered order, unless you define them again in your own CSS.

A very barebones CSS reset could be the following:

* { margin: 0; padding: 0; }

This format would allow you to get rid of all the padding and the margin of all elements but all the other characteristins that are preset in the browser (font family and size, vertical align, borders etc) would stay the same. If you want to use some more complicated reset option, you can do an online search and find several useful pages that provide different options on how to reset the CSS in the browser. Finally, there is even a library called normalize.css that does the job for you if you decide to install it in your application.
