Responsive design

In this page, there were changes only on the elements that needed modification in order to render properly for the mobile enviromnent. These elements follow the usual CSS rules but they are all wrapped in the @media rule. The media rule is used to define different styles for different types of screen sizes.

In order to set the proper dimensions for all of the elements, we need to take under consideration the dimensions of the device being used. In addition, the width of a device and the viewport width of the device are different. Hence, when we use the width attribute, we are referring to the device's width and not the width of the viewport (screen).

Additional elements that need to be usually adjusted, are:

  • The size of all the fonts
     h1 {font-size: 180%;} 
  • The display of all the container tags that are usually set on "block":
    div {display: block;}
  • The display of hidden elements like navigation bars:
    nav {display: hidden;}
  • The position of the elements in relation to the interactivity of the user:
    nav:hover {position: absolute; display:block;}

How was CSS used

  • We set up the navigation tag with a display "none" in order to have it hidden.
  • We set up the navigation bar to display "block" during the hover / click action.
  • We set all the div and section tags in a display "block".
  • We set up the navigation icon to float left and the logo icon to an absolute position.
  • We modified te margins of different text elements to properly position and render in the page with each other.
  • We modified the footer's size and line height in order to not occupy too much space
