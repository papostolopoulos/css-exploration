Specificity


Specificity is useful to understand because it helps us set priorities when two different elements share conflicting properties.


CSS changes at a tag level


In this paragraph and all paragraphs that follow, we have changed the font size to 20 pixels, the left padding to 10 pixels and the text color to green.
This change replaces the default browser values using the following code:


p {
color: green;
left-padding: 10px;
font-size: 20px;
}


CSS changes at a Class level


While some paragraphs below have been given the same css property values as the opening paragraph at the tag level,
they are displayed differently on the page because they have also been given classes.
These paragraphs have been given a class named "class-example1":

<p class="class-example1">

The values of these classes have priority over all values at the tag level. The text color is now larger and blue (30 pixels). Likewise, any other paragraphs that are also defined with the same class are rendered in the same way.


This paragraph has reverted back to the original <p> tag formatting. Since it does not have a class assigned to it, the content renders based on the values given to it at the <p> tag level.


Now, watch how the cascade works. Take a look at the code below. Notice how there are conflicting property values given to the class named "class-example2".


p {
.class-example2 {
color: blue;
}

.class-example2 {
color: yellow;
}

.class-example2 {
color: red;
}

In this example, we first define the class as having a font color blue.
We then alter the cascade by setting the class's color to yellow.
Finally, we change the color to red.
Because the font color red was the last value defined in the cascade, it took precedence over green and yellow.
This is an example of how the properties of an element can be altered downward in the cascade by resetting property values.


CSS changes at an ID level


We have now used another selector, the id, to change this paragraph to purple and the font size to 20 pixels.
Even though this paragraph also has a class that defines the font color as red, it is purple because of the unique ID.


#id-example1 {
color: purple;
font-size: 20px;
}

Summary


We have now used another class and no id for this and the following paragraphs.

p {
.class-example3 {
font-size: 20px;
color: black;
}

In the examples shown above, we obtain specificity in our style sheets by different methods.
Once a style is defined for an element, it will persist
unless defined otherwise in a CSS selector that occurs later on the stylesheet.


Second, we have a heirachy of precedence.
When using just the element as the selector in our CSS, we have assigned the lowest amount of weight for that style.
Classes have more weight than element tags when used as selectors, and therefore, the values defined by their properties will take precedence over that of those defined for the element tags.


Lastly, ID's take precedence over classes, but also must be unique to a single element tag.
