Box Model

It's important for any developer to understand the box model so that they can understand exactly how much space a given html element will take up on a web page. An html element's size is not as simple as the content's defined height and width. Rather, an html element's size is defined by the sum of that element's content, padding, border and margin.

The css properties that we can define when we take the box model under consideration in deciding the size of an element are:

  • The width:
     div {width: 100px} 
  • The height:
     div {height: 50%} 
  • The padding:
     div {padding: 0 10px} 
  • The margin:
     div {margin: 0 auto} 
  • The borders:
     div {border: 2px solid black} 
  • The border's radius:
     div {border-radius: 20px} 

In this manner, we can accomplish a better positioning of the page's different elements.


The box sizing

Even though the box model helps us define the precise size of all of our page elements, it can be challenging and quite confusing because it requires us to calculate the total size of the element based on the sum of its margin, border, padding and width.

This can be resolved by using the "content-box" property. What this property does is that it maintains the width of an element regardless of the extra padding, borders and margin that might be there.


How was CSS used on this page?


  • -We followed the box sizing model in order to avoid any sizing leakage in the document
  • -We added width, padding and borders in the different elements of the page in order to guarantee enough space between elements and better readability.
